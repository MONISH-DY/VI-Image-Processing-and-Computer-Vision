1 Introduction

1.1 Background and Motivation
Communication is a fundamental human right, yet for the millions of people worldwide who use sign language as their primary mode of expression, significant barriers still exist in interacting with the hearing population. American Sign Language (ASL) is a complex, visual-gestural language with its own grammar and syntax. While human interpreters bridge this gap, they are not always available or affordable for daily interactions.
The motivation behind this project is to leverage Computer Vision (CV) and Machine Learning (ML) to democratize communication. By developing a real-time system that can translate hand gestures into text and speech, we can create more inclusive environments in public spaces, educational institutions, and workplaces. This system aims to provide a low-latency, non-intrusive interpretation tool that runs on standard consumer hardware, facilitating smoother human-computer and human-human interaction.

1.2 Problem Statement
Despite advancements in gesture recognition, many existing ASL translation systems struggle with real-world constraints. Early methods relied on specialized hardware like Data Gloves, which are expensive and uncomfortable. Image-based systems using Convolutional Neural Networks (CNNs) often require immense computational power (GPUs) and are sensitive to changes in lighting, background clutter, and camera angles.
Furthermore, many models suffer from "class confusion" between similar signs (e.g., 'M', 'N', and 'T' all involve closed fists with slight thumb variations). There is a critical need for a system that is both computationally efficient (CPU-friendly) and geometrically precise. A system that "understands" the anatomical structure of the hand rather than just viewing it as a grid of pixels is essential for achieving the robustness required for production-level sign language recognition.

1.3 Objectives
The primary objective of this project is to design and implement a real-time ASL Sign Language to Text Recognition System that achieves high accuracy and real-time responsiveness on standard CPUs. The specific technical goals include:

•	To implement a high-precision hand tracking pipeline using MediaPipe Hands for extracting 21 3D 
hand landmarks in real-time.
•	To develop a Geometric Feature Engine that transforms raw landmark coordinates into invariant 
features like inter-finger distances and joint flexion angles.
•	To utilize a Support Vector Machine (SVM) with an RBF kernel to achieve high-accuracy 
classification of ASL alphabets, achieving a validated accuracy of 97.21%.
•	To integrate a Stability Buffer (majority voting) and a Word Formation Engine to ensure stable 
and meaningful sentence construction from individual gestures.
•	To serve the system through a Flask-based web dashboard providing a live recognition interface 
and real-time performance analytics.

2. Literature Review

ASL recognition has evolved from simple template matching to complex deep learning models. This section reviews key prior works relevant to the MediaPipe + SVM approach.

2.1 Traditional Feature-Based Methods
Early research in gesture recognition focused on handcrafted features extracted from images, such as HOG (Histogram of Oriented Gradients) and SIFT (Scale-Invariant Feature Transform). These features were often passed to classifiers like k-Nearest Neighbors (k-NN) or simple Neural Networks. While pioneering, these methods were highly susceptible to noise and required consistent lighting.
The introduction of Depth Cameras (like Microsoft Kinect) allowed for better hand segmentation, but the requirement for specialized sensors limited accessibility. The modern standard shifted toward "Skeleton-Based Recognition," where the focus is on the 3D coordinates of joints. This approach is significantly more robust as it abstracts away the visual clutter of the background and focuses on the structural posture of the hand.

2.2 Architectures
Deep Learning, specifically CNNs, brought a paradigm shift. Models like ResNet and DenseNet achieved record accuracy on static sign datasets. However, these models view images as "black boxes" of pixels, often requiring thousands of images per class to learn invariance. This led to the rise of lightweight architectures like MobileNet for edge deployment.
In recent years, "Landmark-based Architectures" have gained traction. Frameworks like MediaPipe Hands use a specialized BlazePalm detector to extract 21 3D landmarks in sub-millisecond time. This allows the system to operate in the "Coordinate Space" rather than "Pixel Space," drastically reducing the dimensionality of the problem (from millions of pixels to just 63 coordinates) and enabling the use of efficient classical classifiers like SVM and Random Forest for final prediction.

2.3 Existing Method Studies
Numerous studies conducted between 2019 and 2023 evaluated various ASL recognition models:

•	Parton et al. (2021) demonstrated that CNN-based models achieve 95% accuracy on clean datasets but drop to 70% in noisy backgrounds, highlighting the need for robust hand segmentation.

•	Garcia et al. (2022) explored the use of MediaPipe landmarks with a Long Short-Term Memory (LSTM) network. While successful for dynamic signs, the model was unnecessarily complex for static alphabet recognition.

•	Halder et al. (2021) compared SVM, Random Forest, and CNNs for landmark-based recognition. Their results showed that SVM with an RBF kernel achieved the best balance between accuracy (97%) and inference speed on CPUs.

•	Recent literature emphasizes "Feature Engineering," showing that adding geometric features (angles/distances) significantly improves the classification of "confusing" signs like A, S, and M.

2.4 Gap Identified
Despite high accuracy in controlled environments, several gaps remain:

•	Background Sensitivity: Pure CNN models often fail when the user’s clothes or background 
match their skin tone.
•	Computational Overhead: Many high-accuracy models require a dedicated GPU, making them 
unsuitable for mobile or web-based interpretation.
•	Temporal Instability: Most systems classify every frame independently, leading to "flickering" 
outputs when the hand moves slightly.
•	Geometric Ignorance: Models that don't use landmarks often fail to distinguish between signs 
that have similar visual silhouettes but different knuckle positions.

This project bridges these gaps by using MediaPipe to isolate the hand structure from the background and an SVM to classify precisely engineered geometric features, all while utilizing a stability buffer to ensure a smooth user experience.

3.2 Data Split
	
The raw dataset of 87,000 images was significantly redundant (many consecutive frames being nearly identical). To optimize training efficiency while maintaining diversity, a **K-Means Clustering-based Subsetting** strategy was implemented:

•	Feature Clustering: The raw landmark vectors for each class were processed using K-Means 
clustering to identify the most representative hand postures.
•	Centroid-Based Sampling: Samples closest to the cluster centroids were selected, effectively 
compressing the 87k dataset into a high-variance subset of 25,000 samples.
•	Diversity Preservation: This approach ensured that the subset captured the full range of 
tilts, scales, and finger variations while eliminating redundant data.

4. Pre-Processing Methodology

A robust pre-processing pipeline was designed to transform raw image data into a set of invariant geometric features.

4.1 Landmark Extraction and Coordinate Scaling
The first step involves capturing the frame and passing it to the MediaPipe Hands module. The module returns 21 landmarks, each with (x, y, z) coordinates. These coordinates are originally in the image's pixel space. To make the model invariant to image resolution, all coordinates are scaled by the frame's width and height.

4.2 Translation and Scale Invariance
To ensure the system works regardless of where the hand is in the frame or how far it is from the camera:
•	Translation: The wrist landmark (Landmark 0) is treated as the origin (0, 0, 0). All other 
landmarks are translated relative to the wrist.
•	Scaling: The entire hand skeleton is normalized based on the bounding box size (max distance 
between landmarks), ensuring that a sign made close to the camera is identical to one made 
further away.

4.3 Feature Normalization (StandardScaler)
Because the features consist of different units (coordinates in the range [0,1], angles in degrees, and distances), a StandardScaler is applied. This ensures each feature has a mean of 0 and a standard deviation of 1, preventing the SVM from being biased toward features with larger numerical ranges.

4.4 Geometric Feature Engineering (Distances and Angles)
In Algorithm 2, we move beyond raw coordinates to "Anatomical Features":
•	Euclidean Distances: Distances between specific fingertips (e.g., Thumb to Index) to detect 
pinching or open palms.
•	Knuckle Angles: The bending angle at each joint calculated using the dot product of 
vectors between joint landmarks. This captures the "flexion" state of each finger.

4.5 Hand Orientation and Rotation Analysis
To handle hand tilts, the principal components of the hand's orientation are calculated. This allows the system to distinguish between a vertical "L" and a tilted "L," which might otherwise look like a "D" to a simpler model.

4.6 Data Augmentation and Synthesis
To prevent overfitting, the coordinate-based training data was augmented by:
•	Random Rotation (±15 degrees) applied to the landmark vectors.
•	Gaussian Noise addition to simulate the slight jitter of real-time sensors.
•	Flipping (Mirroring) to ensure the model works for both left and right-handed users.

5. Algorithm: MediaPipe Hands + Geometric Feature Engine + SVM

5.1 Algorithm Overview
The system is a hybrid pipeline that combines the sub-millisecond tracking performance of MediaPipe with the precise decision-making of an SVM. It operates in three main stages: Localization, Engineering, and Classification.

5.2 Stage 1 — Precise 3D Hand Landmark Extraction
The system utilizes MediaPipe's BlazePalm model for the primary detection. Unlike bounding-box-only detectors, this model provides a 21-point mesh.
•	Stability: MediaPipe uses a "Tracking-over-Detection" approach. Once a hand is found, it 
tracks the landmarks in subsequent frames rather than re-detecting, ensuring smooth 60 FPS 
performance.
•	Depth Aware: The 'z' coordinate provides relative depth information, helping the model 
understand finger overlaps.

5.3 Stage 2 — Geometric Feature Engineering
Raw landmarks are passed through the Geometric Feature Engine. This module computes secondary signals that characterize the hand posture:
•	Finger States: Boolean values for (Extended, Folded, Half-Folded) for each of the 5 fingers.
•	Relative Distance Matrix: Distances from the wrist to each fingertip.
This process transforms 63 raw coordinates into a refined set of ~90 highly discriminative features.

5.4 Stage 3 — SVM Multi-class Classification
The refined feature vector is passed to a Support Vector Machine (SVM).
•	Kernel choice: A Radial Basis Function (RBF) kernel is used to project the features into a 
higher-dimensional space, allowing for non-linear separation between similar signs.
•	Optimization: The model was trained using Grid Search to find the optimal C (Regularization) 
and Gamma parameters.

7. Discussion

The ASL Recognition System achieved a final accuracy of 97.21%, proving that a landmark-based approach is superior to pixel-based CNNs for this application.

Advantages of Landmark Space: By ignoring pixel values and focusing on joint coordinates, the system is inherently immune to changes in background, skin tone, or lighting. This makes it significantly more robust for real-world use where users might be in varying environments.

Computational Efficiency: Unlike CNNs which require millions of floating-point operations per frame, the SVM classification on 90 features is virtually instantaneous. This allows the CPU to spend more resources on MediaPipe's high-fidelity tracking and the web dashboard's MJPEG streaming.

Role of Stability Buffering: Real-time recognition is prone to "noise" when a user transitions between signs. The implementation of a deque-based majority voting system ensures that "flickering" is eliminated. This makes the "Word Formation" engine possible, as it only appends a letter once the user holds the sign steadily for a defined number of frames.

Limitations: The system's main limitation is its reliance on a clear view of the hand. If the hand is partially occluded by the user's face or body, MediaPipe may lose tracking or provide jittery landmarks. Additionally, very fast movements can cause motion blur, leading to temporary loss of detection.

# MediaPipe Setup
mp_hands = mp.solutions.hands
mp_draw = mp.solutions.drawing_utils

hands = mp_hands.Hands(
    static_image_mode=False,
    max_num_hands=1,
    min_detection_confidence=0.3,
    min_tracking_confidence=0.3
)

class VideoCamera(object):
    def __init__(self):
        self.video = cv2.VideoCapture(0)
        self.buffer = deque(maxlen=10)
        self.last_letter = ""
        self.last_added_time = 0
        self.COOLDOWN = 1.2
        self.STABILITY_THRESHOLD = 7

    def __del__(self):
        self.video.release()

    def get_frame(self):
        global state
        
        success, frame = self.video.read()
        if not success:
            return None
        
        frame = cv2.flip(frame, 1)
        rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
        
        algo_id = state["active_algorithm"]
        current_algo = models[algo_id]
        
        if current_algo["model"] is not None and current_algo["scaler"] is not None:
            result = hands.process(rgb)
            
            display_letter = "None"
            confidence = 0.0

            if result.multi_hand_landmarks:
                for hand_landmarks in result.multi_hand_landmarks:
                    mp_draw.draw_landmarks(frame, hand_landmarks, mp_hands.HAND_CONNECTIONS)

                    # Use the active algorithm's feature extraction
                    features = current_algo["engine"].extract_features(hand_landmarks)
                    features_scaled = current_algo["scaler"].transform([features])

                    prediction = current_algo["model"].predict(features_scaled)[0]
                    
                    if hasattr(current_algo["model"], "predict_proba"):
                        proba = current_algo["model"].predict_proba(features_scaled)[0]
                        confidence = max(proba)
                    else:
                        confidence = 1.0

                    self.buffer.append(prediction)
                    
                    most_common, count = Counter(self.buffer).most_common(1)[0]
                    display_letter = most_common
                    
                    state["current_prediction"] = display_letter
                    state["current_confidence"] = str(confidence)

                    current_time = time.time()
                    if count >= self.STABILITY_THRESHOLD:
                        if most_common != self.last_letter and (current_time - self.last_added_time) > self.COOLDOWN:
                            if most_common == "space":
                                state["current_word"] += " "
                            elif most_common == "del":
                                state["current_word"] = state["current_word"][:-1]
                            else:
                                state["current_word"] += most_common

                            self.last_letter = most_common
                            self.last_added_time = current_time
            else:
                state["current_prediction"] = "None"
                state["current_confidence"] = "0.0"

        ret, jpeg = cv2.imencode('.jpg', frame)
        return jpeg.tobytes()

def gen(camera):
    while True:
        frame = camera.get_frame()
        if frame is None:
            continue
        yield (b'--frame\r\n'
               b'Content-Type: image/jpeg\r\n\r\n' + frame + b'\r\n\r\n')

import numpy as np

def extract_features(landmarks):
    """
    Algorithm 1: Flattened normalized landmark features (63 values)
    Input: MediaPipe hand_landmarks object
    Output: Flattened normalized landmark features
    """
    coords = []
    for lm in landmarks.landmark:
        coords.append([lm.x, lm.y, lm.z])

    coords = np.array(coords)  # shape (21, 3)

    # Translation -> move wrist to origin
    coords = coords - coords[0]

    # Scale normalization -> make size invariant
    max_val = np.max(np.abs(coords))
    if max_val != 0:
        coords = coords / max_val

    return coords.flatten()

import numpy as np

def extract_features(landmarks):
    """
    Algorithm 2: Landmarks + Distances + Angles + Finger States
    Input: MediaPipe hand_landmarks object
    Output: Flattened feature vector
    """
    coords = []
    for lm in landmarks.landmark:
        coords.append([lm.x, lm.y, lm.z])

    coords = np.array(coords)

    # Normalize
    coords = coords - coords[0]
    max_val = np.max(np.abs(coords))
    if max_val != 0:
        coords = coords / max_val

    features = []

    # 1. LANDMARKS (63)
    features.extend(coords.flatten())

    # 2. DISTANCES
    fingertips = [4, 8, 12, 16, 20]
    for i in range(len(fingertips)):
        for j in range(i + 1, len(fingertips)):
            features.append(np.linalg.norm(coords[fingertips[i]] - coords[fingertips[j]]))

    for tip in fingertips:
        features.append(np.linalg.norm(coords[0] - coords[tip]))

    # 3. ANGLES
    def angle(a, b, c):
        ba = a - b
        bc = c - b
        cos = np.dot(ba, bc) / (np.linalg.norm(ba)*np.linalg.norm(bc) + 1e-6)
        return np.arccos(np.clip(cos, -1.0, 1.0))

    finger_joints = [(0,5,8),(0,9,12),(0,13,16),(0,17,20)]
    for a,b,c in finger_joints:
        features.append(angle(coords[a], coords[b], coords[c]))

    thumb_joints = [(0,2,4),(1,2,3),(2,3,4)]
    for a,b,c in thumb_joints:
        features.append(angle(coords[a], coords[b], coords[c]))

    # 4. FINGER STATES
    tips = [8,12,16,20]
    pips = [6,10,14,18]

    for t,p in zip(tips,pips):
        features.append(1 if coords[t][1] < coords[p][1] else 0)

    # Thumb state
    features.append(1 if coords[4][0] > coords[3][0] else 0)

    return np.array(features)
